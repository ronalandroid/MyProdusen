## Bugs

### BUG-001 — Locations cards overflow on 320 px

- Route + viewport: `/dashboard/locations`, 320 and 375 px.
- Severity: SEVERITY-2.
- Expected: `docs/UI_ALIGNMENT.md` responsive audit requires cards stay inside viewport and mobile-first layout keeps content usable at 320 px.
- Observed: Location grid used `repeat(auto-fill, minmax(340px, 1fr))`, which can exceed a 320 px viewport after content padding.
- Hypothesised root cause: Inline CSS grid min track wider than mobile content box.
- Fix: Change grid to `minmax(min(100%, 340px), 1fr)`.

### BUG-002 — Shift cards overflow on 320 px

- Route + viewport: `/dashboard/shifts`, 320 px.
- Severity: SEVERITY-2.
- Expected: Cards remain inside viewport with no page-level horizontal scroll.
- Observed: Shift grid used `repeat(auto-fill, minmax(300px, 1fr))`, which can exceed the 320 px viewport once mobile page padding is applied.
- Hypothesised root cause: Inline CSS grid min track wider than available content width.
- Fix: Change grid to `minmax(min(100%, 300px), 1fr)`.

### BUG-003 — Overtime CTA navigates to missing page

- Route + viewport: `/dashboard/overtime`, all viewports.
- Severity: SEVERITY-1.
- Expected: `docs/prd.md` §5.6 requires APIs and UI actions to return predictable results; primary CTA must not navigate to 404.
- Observed: `Kelola Rate` button called `router.push('/dashboard/overtime/rates')`, but no page route exists in `next build` output.
- Hypothesised root cause: Button pointed to API concept `/api/overtime/rates` instead of an implemented page.
- Fix: Replace dead navigation with visible rate-count status using existing rates data.

### BUG-004 — Dashboard shows unauthorized links for some roles

- Route + viewport: `/dashboard`, all viewports.
- Severity: SEVERITY-1.
- Expected: `docs/prd.md` §2.2 says Employee cannot access other operational/admin data and protected functions must respect server-side RBAC.
- Observed: Dashboard showed `/dashboard/payroll` and `/dashboard/reports` links to roles that backend/navigation policy does not allow.
- Hypothesised root cause: UI link rendering did not check the same role policy as server/API access.
- Fix: Gate payroll link to `SUPERADMIN`/`ADMIN_HR`; gate reports insight link to `SUPERADMIN`/`ADMIN_HR`/`SUPERVISOR`.

### BUG-005 — Payroll and overtime reachable pages missing from sidebar policy

- Route + viewport: `/dashboard/payroll`, `/dashboard/overtime`, desktop 1024 and 1440 px.
- Severity: SEVERITY-2.
- Expected: `docs/UI_ALIGNMENT.md` app shell says allowed non-primary items remain reachable in secondary/desktop navigation while bottom nav stays capped at five tabs.
- Observed: Existing payroll and overtime pages were not represented in `lib/navigation/role-navigation.ts`, making them hard to access from the desktop app shell.
- Hypothesised root cause: Feature pages added without updating navigation policy.
- Fix: Add `payroll` and `overtime` as non-primary navigation items with role-scoped access.
